# Обзор

## Описание
Секция описывает концепцию перехода от текущей архитектуры с двумя отдельными бинарниками (nexbot для основного процесса и subagent для контейнеризированных задач) к единому бинарнику с подкомандами. Также описывается использование контейнеризации на базе alpine вместо кастомных Docker-образов.

## Ключевые элементы
- **Единый бинарник**: Объединение nexbot и subagent в один исполняемый файл с подкомандами (serve, subagent)
- **Alpine-контейнеры**: Замена кастомного Docker-образа `nexbot/subagent` на стандартный alpine:3.23 (~5MB)
- **Volume mounts**: Бинарник и конфигурация монтируются с хоста, не требуется пересборка образа

## Практические выводы
1. Не требуется пересборка Docker-образа при изменениях кода — только пересборка бинарника
2. Упрощается деплой: один бинарник вместо двух + Docker-образ
3. Уменьшается время CI/CD за счёт исключения этапов docker build и push
